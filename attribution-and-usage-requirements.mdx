---
title: "Attribution & Usage Requirements"
---

When using the ArcWeather API or any ArcWeather-provided data, you are required to provide clear and visible attribution to ArcWeather.

This ensures proper credit, transparency for end users, and alignment with ArcWeather’s usage policies.

## Required Attribution

All applications, websites, or services using ArcWeather data must include attribution in one of the following ways:

### Standard Attribution

At minimum, include the following text:

> **Data provided by ArcWeather**

This text must link to:

```text
https://arcweather.cloud
```

## Placement Guidelines

### Web Pages & Applications

- Attribution must be **clearly visible on the page** where ArcWeather data is displayed
- It should not be hidden in footers that are not visible or behind interactions
- It should be reasonably close to the data being shown

### Maps & Interactive Displays

If ArcWeather data is used within a map:

- Attribution must be placed within the **map attribution control** or equivalent UI
- Example:

> Data provided by ArcWeather

(linked to [https://arcweather.cloud](https://arcweather.cloud))

### Static Outputs (Images, Reports, Emails)

For static or exported content:

- Include attribution **directly on the asset** or within the surrounding content
- Examples:
  - Image watermark or caption
  - Email footer
  - Report footer

## Required Notice

In addition to attribution, you must include a notice that clarifies the intended use of ArcWeather data.

### Recommended Notice

> ArcWeather data is provided for informational purposes only and should not be used for life-saving or emergency decision-making.

This notice should be:

- Clearly visible
- Located near the data or within the same interface
- Included in any application where decisions could be influenced by weather data

---

## Limitation of Liability

You must also communicate that ArcWeather is not liable for decisions made based on the data.

### Recommended Language

> ArcWeather makes no guarantees regarding the accuracy or completeness of the data and is not responsible for any decisions made based on this information.

## Additional Guidelines

- Do not remove, obscure, or alter attribution
- Do not present ArcWeather data as your own
- Do not imply ArcWeather endorses your application or use case
- Ensure attribution remains visible across all screen sizes and devices

## Summary

- Attribution to ArcWeather is **required for all uses**
- Must include a **link to [arcweather.cloud](http://arcweather.cloud)**
- Must be **visible and appropriately placed**
- Must include a **non-life-saving data notice**
- Must include a **limitation of liability statement**
